1. 引言
1.1 产品背景
在当代社会，年轻人面临诸多不确定性，对自我认知、情感指导、个人成长的诉求日益增强。传统命理（八字、黄历）与现代心理学（MBTI）、占星学（星座）各自拥有庞大的受众，但缺乏有机融合。同时，AI技术的发展使得个性化、实时性的内容生成成为可能。本产品旨在通过科技感的视觉呈现（Canvas粒子八卦图），融合东西方智慧，为用户提供兼具娱乐性与实用性的每日指引、姻缘配对、目标管理与弱点克服服务。

1.2 产品愿景
成为用户每日必开的“自我成长伴侣”——通过科技与玄学的结合，帮助用户更了解自己，更从容地面对生活挑战，在趣味中实现个人进步。

1.3 核心目标
提高用户留存：通过每日更新的个性化内容（黄历、指引、练习）和习惯养成机制，让用户形成每日打开的习惯。

多元化变现：通过订阅、按次付费、增值服务等方式实现商业闭环。

打造品牌特色：以炫酷的粒子特效、深度的多维数据融合、AI生成内容为核心竞争力，区别于传统运势类App。

2. 产品概述
2.1 产品定位
一款融合八字、MBTI、星座、黄历的AI驱动型个人成长工具，以科技感视觉呈现，提供每日指引、姻缘配对、目标管理、弱点克服等功能。

2.2 目标用户
Z世代年轻人（18-28岁）：对星座、MBTI有浓厚兴趣，追求新鲜事物，愿意为情绪价值付费。

职场白领（25-35岁）：面临事业、感情压力，需要情感陪伴和行动建议，消费能力较强。

传统文化爱好者：对八字、黄历有一定了解，希望获得更便捷的工具和深度解读。

2.3 核心价值
个性化：所有内容基于用户真实信息（出生时间、MBTI）生成，千人千面。

融合性：将东西方多个符号体系交叉分析，提供复合型洞察。

行动导向：不只是预测，更给出每日可执行的行动建议，帮助用户改善生活。

科技感体验：Canvas粒子特效构建的动态八卦图，让每一次交互都充满仪式感。

3. 功能需求
3.1 用户档案模块
描述：记录用户的基本信息，作为所有个性化内容的基础。

功能	说明
出生信息录入	公历出生日期（年月日时），用于八字排盘和星座计算
MBTI测评	提供两种模式：快速选择（直接选类型）和对话式测评（通过8轮问答自动推断）
信息编辑	允许用户修改出生信息（需重新计算关联数据）
多用户支持	可添加亲友档案，用于配对等功能（本地存储）
数据字段：

用户ID（本地生成）

出生公历日期

出生时辰（可选，无则默认子时）

MBTI类型

自定义昵称

3.2 首页与粒子特效
描述：App首页以Canvas粒子构成动态八卦图，作为核心视觉交互区。

功能	说明
粒子八卦图	粒子数量200-500，可构成太极图轮廓，缓慢旋转
动态响应	根据当日运势、用户操作（如打卡）改变粒子颜色、速度、大小
底部导航	包含“首页”、“目标”、“成长”、“我的”等Tab
今日卡片	首页悬浮半透明卡片，展示每日黄历摘要、一句AI箴言、今日待办提醒
3.3 每日指引模块
描述：每日为用户生成融合八字、MBTI、星座、黄历的个性化指引。

功能	说明
自动生成	每日首次打开时触发，或用户手动刷新
内容结构	包含今日黄历（宜忌）、基于用户画像的运势解读、1-3条具体行动建议
分享功能	可将今日指引生成长图分享至社交平台
历史回顾	可查看过往指引记录
AI生成逻辑：结合结构化数据（八字日运、MBTI类型、星座运势、黄历宜忌）输入大模型，生成自然语言解读。

3.4 姻缘配对模块
描述：支持单人“理想型”分析和双人配对深度报告。

功能	说明
理想型分析	基于用户自身信息，推荐最匹配的MBTI、星座、八字特征
双人配对	输入两人的出生信息/MBTI，生成综合匹配度和详细报告
配对码分享	生成加密配对码，对方扫码即可快速配对（仅交换必要信息）
报告内容	包含八字合婚评分、MBTI契合度、星座匹配度、AI融合解读（缘起/相处/指引）
付费点	深度报告（含流年分析、吉日建议）需付费解锁
3.5 目标管理模块
描述：帮助用户设定个人目标，每日获得基于多维信息的激励指引，并记录进度。

功能	说明
新建目标	输入名称、类型（学习/健康/事业/感情/其他）、周期（7/21/30天/自定义）、每日计划
目标列表	展示所有进行中的目标，进度条、剩余天数、今日打卡状态
每日指引	每个目标每日生成一条个性化行动建议（融合黄历、用户画像）
打卡记录	用户可标记完成度（0-100%），系统记录历史，更新进度条
粒子联动	根据整体完成度调整粒子特效（颜色、速度、大小）
阶段复盘	每周/月生成AI复盘报告，总结进展并给予鼓励
3.6 弱点克服模块
描述：针对用户选择的弱点（如执行力差、拖延、情绪化），每日生成克服练习，帮助用户自我提升。

功能	说明
弱点选择	提供常见弱点库（执行力差、拖延、焦虑、易怒、优柔寡断等），支持自定义
多维分析	从八字、MBTI、星座角度解释弱点成因，让用户理解“为什么我有这个弱点”
每日练习	每日生成1-2条具体行动练习（如“上午10点专注25分钟”），融合当日黄历
练习打卡	用户完成练习后可打卡，记录连续天数
成就系统	达成里程碑（连续7天、30天）可获得徽章，解锁特殊粒子效果
数据联动	与目标管理模块联动，练习可关联到具体目标
3.7 个人中心与设置
功能	说明
用户档案	查看/修改出生信息、MBTI
订阅管理	查看当前订阅状态、购买记录、续费
历史记录	查看过往每日指引、配对报告、目标复盘
隐私设置	数据存储方式（仅本地/云同步）、匿名模式
反馈与帮助	提交问题、常见问题解答
4. 非功能需求
4.1 性能要求
粒子系统：在主流中端手机上保持60fps流畅渲染，粒子数量建议200-300（可动态调整）。

AI请求：首次加载卡片时，若需调用AI，应显示加载动画，超时（5秒）则降级显示结构化数据。

冷启动：App启动时间不超过2秒。

4.2 安全与隐私
数据本地化：所有用户个人信息（出生时间、MBTI）默认仅存储在本地（IndexedDB/localStorage）。

云同步可选：如需跨设备使用，需用户主动开启并同意加密存储（使用AES-256加密）。

配对码：仅包含必要信息（如星座、MBTI、简略五行），不暴露完整八字。

AI请求匿名化：调用大模型API时，不传输用户真实姓名等敏感信息，可使用随机ID代替。

4.3 兼容性
iOS Safari / Android Chrome / 微信内置浏览器（PWA版本）

响应式设计，适配手机、平板等不同屏幕尺寸。

6. AI集成设计
6.1 大模型选择
推荐使用 DeepSeek Chat 或同类大模型 API，具备良好的中文生成能力和较低的成本。

6.2 提示词工程
为每个功能模块设计专门的提示词模板，确保生成内容的准确性和多样性。提示词应包含结构化数据（如八字五行分数、MBTI特质、星座元素、黄历宜忌）并要求按照固定格式输出。

示例（每日指引提示词）：

text
你是一位融合东西方智慧的成长教练。请根据以下信息生成今日指引：
【用户】MBTI: INTJ (理性, 战略), 星座: 处女座, 八字日主: 癸水
【今日天时】公历2026-03-11, 农历二月初二, 干支丙午 辛卯 壬戌, 宜祭祀/求财, 忌嫁娶/出行
【要求】1. 先写一句今日核心能量；2. 分两段写运势解读与行动建议；3. 结尾一句鼓励。
6.3 缓存策略
每日指引、配对报告等生成后缓存至本地，相同内容不再重复请求API。

设置有效期（如每日指引有效期至次日凌晨），过期后重新生成。

6.4 降级方案
当API请求失败或超时时，前端应基于结构化规则生成简版内容，避免无数据显示。

7. 交互与视觉设计
7.1 视觉风格
主色调：黑（背景）、金（粒子、高亮）、青（光晕、点缀）

字体：科技感无衬线字体（如Poppins、Roboto），标题可使用特殊字体增强神秘感

粒子效果：粒子带发光、拖尾效果，根据用户状态变化

7.2 关键交互
首页进入：粒子从中心扩散形成八卦图，同时今日卡片淡入

切换Tab：粒子短暂加速后平稳过渡

打卡：点击打卡按钮，粒子向外扩散光波，卡片翻转显示“已完成”

配对生成：两人信息输入后，粒子分裂成两个漩涡，逐渐融合或分离反映匹配度

7.3 动效原则
所有动效应流畅自然，不干扰信息获取

粒子变化与用户操作强关联，提供即时反馈

避免过度动效导致性能问题

9. 风险与合规
9.1 法律风险
明确产品定位为“娱乐工具”和“个人成长助手”，而非算命平台。

在每份报告末尾显著位置添加免责声明：“内容由AI生成，仅供娱乐参考，请勿作为决策依据。”

避免使用“大凶”、“血光之灾”等恐吓性词汇，引导积极正向。

9.2 数据隐私
遵守《个人信息保护法》，用户数据仅用于本产品，不上传至第三方。

云同步需用户主动开启，并明确告知数据加密存储。

9.3 内容审核
AI生成内容需经过敏感词过滤，确保不包含政治敏感、色情、暴力等内容。

建立人工审核机制，定期抽查生成内容。

5. AI Coding 落地建议 (How to prompt AI)

如果你在用 Cursor，请直接在 .cursorrules 中加入以下逻辑限制，能让 AI 帮你写出更稳的代码：

Rule 1 (Data): “始终使用 lunar-javascript 处理农历逻辑，不要尝试自写算法。”
Rule 2 (UI): “所有的 UI 组件必须包含 framer-motion 的交互反馈，色彩配置参考 tailwind.config.js 中的金/黑/青色调。”
Rule 3 (Local-First): “优先使用 localStorage 存储用户档案，只有在 syncEnabled 为 true 时才考虑 API 同步。”
